# Google Sheets Template

Reference for the exact sheet structure FIRE expects. The `setupSheets` function creates the Categories and Result sheets automatically, but this documents the full structure for reference.

## Sheet Overview

| Sheet | Purpose | Created By |
|---|---|---|
| `Monzo` | Raw Monzo transactions | You (manual) |
| `Revolut` | Raw Revolut transactions | You (manual) |
| `Yonder` | Raw Yonder transactions | You (manual) |
| `Categories` | Category definitions | `setupSheets` |
| `Result` | Normalised + categorised output | `setupSheets` |

## Bank Source Sheets

These are where you paste raw transaction data. Create only the ones you need.

### Monzo

| Column | Header | Example |
|---|---|---|
| A | Transaction ID | `tx_00009...` |
| B | Date | `2024-01-15` |
| C | Time | `14:30:00` |
| D | Name | `Tesco Stores` |
| E | Amount | `-45.67` |
| F | Currency | `GBP` |
| G | Type | `card_payment` |
| H | Category | `groceries` |
| I | Notes and #tags | `weekly shop #groceries` |

- **Transaction ID** is Monzo's native ID — used for deduplication
- **Amount** is negative for debits, positive for credits
- **Category** column from Monzo is ignored by FIRE (AI does its own categorisation)

### Revolut

| Column | Header | Example |
|---|---|---|
| A | ID | *(backfilled by system)* |
| B | Started Date | `2024-01-15 14:30:00` |
| C | Completed Date | `2024-01-15 14:30:05` |
| D | Description | `Tesco` |
| E | Amount | `-45.67` |
| F | Currency | `GBP` |
| G | Type | `CARD_PAYMENT` |

- **ID** is generated by FIRE if empty (Revolut CSV exports don't include IDs)
- **Started Date** is used as the transaction date

### Yonder

| Column | Header | Example |
|---|---|---|
| A | ID | *(backfilled by system)* |
| B | Date/Time of transaction | `15/01/2024 14:30` |
| C | Description | `TESCO STORES 1234` |
| D | Amount (GBP) | `45.67` |
| E | Currency | `GBP` |
| F | Debit or Credit | `Debit` |
| G | Category | `Shopping` |
| H | Country | `United Kingdom` |

- All Yonder transactions are in GBP (no currency conversion needed)
- **Category** column from Yonder is ignored
- **Country** is preserved in the Result sheet

## Categories Sheet

Created by `setupSheets` with these columns:

| Column | Header | Type | Description |
|---|---|---|---|
| A | ID | UUID | Stable unique identifier |
| B | Name | Text | Category name (unique among active) |
| C | Description | Text | What transactions belong here |
| D | Examples | Text | Example merchants/descriptions |
| E | Is Active | Boolean | `TRUE` or `FALSE` |
| F | Created At | DateTime | `yyyy-mm-dd hh:mm:ss` |
| G | Modified At | DateTime | `yyyy-mm-dd hh:mm:ss` |

## Result Sheet

Created by `setupSheets`. This is the main output — all normalised and categorised transactions end up here.

| Column | Header | Type | Description |
|---|---|---|---|
| A | ID | UUID | FIRE's internal transaction ID |
| B | Original Transaction ID | Text | ID from bank source (or generated) |
| C | Bank Source | Enum | `MONZO`, `REVOLUT`, or `YONDER` |
| D | Transaction Date | Date | When the transaction occurred |
| E | Transaction Type | Enum | `DEBIT` or `CREDIT` |
| F | Description | Text | Merchant/description from source |
| G | Notes | Text | Notes/tags from source (if available) |
| H | Country | Text | Country (if available) |
| I | Original Amount | Number | Amount in original currency |
| J | Original Currency | Text | ISO 4217 currency code |
| K | GBP Amount | Number | Amount converted to GBP |
| L | Exchange Rate | Number | Rate used for conversion (null for GBP) |
| M | AI Category ID | UUID | Category UUID assigned by AI |
| N | AI Category | Text | Category name assigned by AI |
| O | Confidence Score | Number | AI confidence (0-100) |
| P | Manual Category ID | UUID | Category UUID from manual override |
| Q | Manual Category | Text | Category name from manual override |
| R | Category | Formula | `=IF(Q<>"", Q, N)` — manual takes precedence |
| S | Processing Status | Enum | `UNPROCESSED`, `NORMALISED`, `CATEGORISED`, `ERROR` |
| T | Error Message | Text | Error details (if status is ERROR) |
| U | Created At | DateTime | When row was created |
| V | Last Modified | DateTime | Last update timestamp |
| W | Normalised At | DateTime | When normalisation completed |
| X | Categorised At | DateTime | When categorisation completed |

### Data Validation

The Result sheet has dropdown validation on:
- **Processing Status** (column S): `UNPROCESSED`, `NORMALISED`, `CATEGORISED`, `ERROR`
- **Transaction Type** (column E): `DEBIT`, `CREDIT`

### The Category Formula

Column R (`Category`) uses a formula: if a manual category is set, it shows that; otherwise it shows the AI category. This is the column you should use for analysis, pivots, and reports.

## Formatting

Both Categories and Result sheets have:
- Bold header row with `#e8eaed` background
- Column widths pre-configured for readability
- Date columns formatted as `yyyy-mm-dd hh:mm:ss`
